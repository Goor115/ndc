## Потік даних (Data Flow)

### 1️⃣ Ініціалізація при завантаженні

```
1. Сторінка завантажується
2. App компонент монтується
3. useAuth() хук ініціалізується:
   - Читає localStorage['currentUser']
   - Якщо не знайдено → повертає null → показує LoginPage
4. useStore() хук ініціалізується:
   - Читає localStorage['personnel']
   - Читає localStorage['schedules']
   - Виконує міграцію старих даних
5. Обидва хука встановлюють state
6. Компоненти렌дерються з первинними даними
```

### 2️⃣ Вхід користувача

```javascript
Користувач вводить дані в LoginPage
          │
          ▼
  Кнопка "Увійти" натиснута
          │
          ▼
  handleSubmit() викликається
          │
          ▼
  onLogin(login, password) визивається
          │
          ▼
  useAuth.login() функція:
    ├─ Пошук користувача в USERS
    ├─ Зберігання в localStorage
    ├─ Оновлення currentUser state
    └─ Повертає true/false
          │
          ▼
  window.location.reload() перезавантажує сторінку
          │
          ▼
  App перевідрендерюється, LoginPage більше не видна
```

### 3️⃣ Вибір дати та завантаження графіку

```javascript
selectedDate = dayjs().format('YYYY-MM-DD')
          │
          ▼
rawSchedule = schedules[selectedDate]
          │
          ▼
schedule = {
  cards: rawSchedule?.cards || [],
  statuses: rawSchedule?.statuses || {},
  header: rawSchedule?.header || {},
}
          │
          ▼
busyPersonnel = getBusyPersonnel(selectedDate, schedule)
          │
          ▼
Компоненти рендеряться з цими даними
```

### 4️⃣ Додання завдання

```javascript
<button>+ Додати наряд</button> натиснута
          │
          ▼
addCard() вхилається
          │
          ├─ Отримати сьогодішній графік з schedules
          ├─ Створити новий card об'єкт з ID = Date.now()
          ├─ Добавити до массиву cards
          ├─ Оновити schedules state
          └─ Зберегти в localStorage
          │
          ▼
Компонент переренрдерюється з новою карточкою
          │
          ▼
<MissionCard> відображається для нової карточки
```

### 5️⃣ Оновлення карточки (вибір водія)

```javascript
Користувач вибирає водія в MissionCard
          │
          ▼
setDriver(person) викликається
          │
          ▼
onUpdate({ driver: person }) -> updateCard() викликається
          │
          ├─ Знайти карточку в schedules[selectedDate]
          ├─ Оновити поле driver
          ├─ Оновити schedules state
          └─ save('schedules', schedules)
          │
          ▼
MissionCard перерендерюється з новим водієм
          │
          ▼
DayStats перераховує статистику (экіпаж)
```

### 6️⃣ Розподіл персоналу по відділам

```javascript
Користувач вибирає pessoa в DepartmentPanel
          │
          ▼
addPersonToDept(dept_key, person) викликається
          │
          ├─ Отримати поточних людей в відділі
          ├─ Додати нову персону
          ├─ Оновити schedules[selectedDate].statuses
          └─ Викликати onUpdate()
          │
          ▼
DepartmentPanel перерендерюється з новою персоною
          │
          ▼
DayStats перераховує (хворі/відпустка/вихідні)
          │
          ▼
Зберігається в localStorage['schedules']
```

### 7️⃣ Видалення персоналу

```javascript
Адмін видаляє person із списку
          │
          ▼
removePersonnel(personId) викликається
          │
          ├─ Видалити з personnel state
          ├─ Проходимо по всіх графіках
          │  ├─ Видалити з карточок (driver, crew)
          │  └─ Видалити з відділів (statuses)
          └─ Зберегти обидва в localStorage
          │
          ▼
Всі компоненти перерендеряються без цієї персони
```

### 8️⃣ Копіювання з попереднього дня

```javascript
Користувач натискає кнопку "Копіювати"
          │
          ▼
copyFromPreviousDay(selectedDate) викликається
          │
          ├─ Получить попередній день
          ├─ Скопіювати schedule об'єкт
          ├─ Генерувати нові ID для карточок
          ├─ Очистити статуси (не копіюємо хворих і т.д.)
          ├─ Очистити заголовок
          └─ Оновити schedules state
          │
          ▼
Нова дата показує скопійовані карточки
          │
          ▼
Користувач може редагувати без впливу на попередній день
```

## Логіка доступності персоналу

```
┌─────────────────────────────────────────┐
│      Вся база персоналу                 │
│  (personnel.length елементів)           │
└─────────────────────────────────────────┘
                  │
        ┌─────────┼─────────┬──────────┐
        │         │         │          │
        ▼         ▼         ▼          ▼
    Зайняті  На хворих На відпустці  Вільні
    на карт  (sick)   (vidpustka)    (free)
        │         │         │          │
        ├─────────┴─────────┴──────────┤
        │                              │
        ▼                              ▼
    busyPersonnel              availablePersonnel
    (НЕ можна добавити)        (Можна добавити)

При додаванні на карточку:
  if (busyPersonnel.has(id))  → ЗАБОРОНЕНО
  if (!busyPersonnel.has(id)) → ДОЗВОЛЕНО

При додаванні в відділ:
  if (busyPersonnel.has(id))        → ЗАБОРОНЕНО
  if (assignedPersonnel.has(id))    → ЗАБОРОНЕНО
  else                              → ДОЗВОЛЕНО
```

## Часові конфлікти (Time conflicts)

### Логіка перевірки

```
Користувач хочет добавити person на карточку
з часом 07:00-14:00
          │
          ▼
getBusyPersonnelForTime('2024-03-15', '07:00', '14:00')
          │
          ├─ Проходимо по всіх карточках дня
          ├─ Для кожной карточки перевіримо перетин часу
          │  Формула:
          │    cardStart < userEnd AND cardEnd > userStart
          │    (07:00 < 14:00 AND 14:00 > 07:00) → 🔴 КОНФЛІКТ
          │
          └─ Додаємо в busySet тих хто конфліктує


Наслідок: Людина НЕ доступна якщо часи перекриваються
```

### Приклади часових конфліктів

```
Карточка 1: 07:00-14:00  (1-а зміна)
Карточка 2: 14:00-21:00  (2-а зміна)
──────────────────────────────────────
Людина Іван НЕ може бути на обох, бо 14:00 це край

Карточка 1: 07:00-14:00  (1-а зміна)
Карточка 2: 10:00-17:00  (День)
──────────────────────────────────────
КОНФЛІКТ: 10:00-14:00 перекривається
Іван НЕ може на обох

Карточка 1: 21:00-07:00  (Нічна)      ← Скрізь дня
Карточка 2: 07:00-14:00  (1-а зміна)
──────────────────────────────────────
КОНФЛІКТ в 07:00
Іван НЕ може на обох
```

## Безпека та валідація

### Проблеми безпеки (у локальному додатку)

```
❌ Пароль видний в кодбазе (для demo)
❌ localStorage доступний будь-якому скрипту
❌ Нема зберігання на сервері
❌ Нема вебонтикації (JWT/sessions)
❌ Нема логування операцій
```

### Валідація що робиться

```
✅ Перевірка логіну/пароля при вході
✅ Перевірка role перед редаганням (isAdmin)
✅ Try-catch при JSON.parse localStorage
✅ Міграція даних при завантаженні
✅ Фільтрація доступних людей
✅ Запобігання дублюванню в одному відділі
✅ Видалення персони з усіх графіків при видаленні
```

## Production готовність

### Що потрібно для production

```
1. ✅ Backend сервер для збереження даних
2. ✅ Справжня аутентифікація (OAuth, JWT)
3. ✅ API для CRUD операцій
4. ✅ Валідація на сервері
5. ✅ Логування та аудит
6. ✅ Обработка помилок и retry логіка
7. ✅ Синхронізація колаборативних змін
8. ✅ Експорт/Імпорт даних
9. ✅ Role-based access control (RBAC)
10. ✅ Тестування (unit, integration, e2e)
```

### Які файли потребуют змін

```
useAuth.js          → Замінити на реальну аутентифікацію
useStore.js         → Замінити на API запити до backend
App.jsx             → Додати error handling, loading states
Всі компоненти      → Додати error boundaries, retry логіку
localStorage usage  → Видалити або використовувати для cache
```
