## 🔑 Ключові терміни

| Термін | Опис |
|--------|------|
| **Наряд** | Завдання для екіпажу на певну зміну з часовим діапазоном |
| **Зміна (Shift)** | Період роботи (1-а, 2-а, ніч, цілий день) з визначеним часом |
| **Екіпаж (Crew)** | Група людей для виконання наряду |
| **Водій (Driver)** | Основна персона на наряді, обов'язкова |
| **Персонал** | Всі люди в системі з категоріями (crew, crew_driver, local, local_driver) |
| **Статус** | Категорія персонала (хвори, на відпустці, на вихідних, в відділах) |
| **Графік** | Дані для конкретної дати (наряди, статуси, заголовок) |
| **localStorage** | Локальне зберігання браузера для персистенції даних |

## 🤔 Часті питання (FAQ)

### Q: Де зберігаються дані?
**A:** У `localStorage` браузера. Ключі: `currentUser`, `personnel`, `schedules`

### Q: Як додати нову персону?
**A:** Використайте PersonnelManager компонент (адмін-тільки) або викликайте `addPersonnel()` з `useStore`.

### Q: Як запобігти двійному призначенню людини?
**A:** Система автоматично фільтрує зайнятих людей через `getBusyPersonnel()`.

### Q: Як скопіювати графік попереднього дня?
**A:** Викликайте `copyFromPreviousDay(date)` з `useStore`.

### Q: Чи потребується backend?
**A:** Поточна версія працює тільки на клієнті. Для production потрібен сервер.

### Q: Як змінити часи змін?
**A:** Редагуйте функцію `getShiftTimes()` в `MissionCard.jsx`.
