# 📋 Документація проекту NDC - Наряд робіт

## Загальний опис проекту

**NDC** — це React + Vite додаток для управління розпорядженнями та розкладами роботи персоналу. Система дозволяє адміністраторам створювати та управляти умовами роботи, призначати людей на різні завдання та відділи, відстежувати їхню занятість.

**Стек технологій:**

- React 19.2.0
- Vite 7.3.1
- Tailwind CSS 4.2.1
- Day.js (робота з датами)
- dnd-kit (drag and drop для сортування)

## 🔐 `useAuth.js` — Система аутентифікації

**Розташування:** `src/store/useAuth.js`

**Призначення:** Управління входом/виходом користувачів та перевірка прав доступу.

### Основні функції:

#### `USERS` — База користувачів

```javascript
const USERS = [
  {
    id: 1,
    login: 'admin',
    password: 'admin123',
    role: 'admin',
    name: 'Адміністратор',
  },
  {
    id: 2,
    login: 'viewer',
    password: 'view123',
    role: 'viewer',
    name: 'Перегляд',
  },
]
```

- Dois передвизначених користувачі з роллю адміністратора та глядача
- **admin** — має повні права на редагування
- **viewer** — може тільки переглядати (без редагування)

#### `useAuth()` — React Hook з методами:

| Метод                              | Опис                                                                                            |
| ---------------------------------- | ----------------------------------------------------------------------------------------------- |
| `login(loginInput, passwordInput)` | Перевіряє облікові дані та зберігає користувача в `localStorage`. Повертає `true` якщо успішно. |
| `logout()`                         | Видаляє поточного користувача і очищає `localStorage`                                           |
| `currentUser`                      | Об'єкт поточного користувача або `null`                                                         |
| `isLoggedIn`                       | Boolean флаг входу                                                                              |
| `isAdmin`                          | Boolean флаг адміністратора                                                                     |

#### Зберігання даних:

- Користувач зберігається в `localStorage` під ключем `currentUser`
- При перезавантаженню сторінки дані зберігаються

## 📊 `useStore.js` — Управління даними

**Розташування:** `src/store/useStore.js`

**Призначення:** Управління графіками, персоналом, завданнями та логіка присвоєння людей на роботу.

### Структура персоналу:

```javascript
interface Personnel {
  id: number
  name: string
  role: 'crew' | 'crew_driver' | 'local' | 'local_driver'
}
```

Ролі:

- **crew** — Екіпаж
- **crew_driver** — Водій екіпажу
- **local** — Місцевий персонал
- **local_driver** — Місцевий водій

### Структура графіку по датам:

```javascript
interface Schedule {
  [date: string]: {
    cards: MissionCard[]           // Завдання/наряди
    statuses: {
      [department: string]: Personnel[]  // Люди в різних стану
    }
    header: {
      chergovyi: string            // Дежурний
      pislya: string               // Після дежурства
      nachalnik: string            // Начальник
    }
  }
}
```

### Основні методи `useStore()`:

| Метод                                             | Параметри                    | Опис                                     |
| ------------------------------------------------- | ---------------------------- | ---------------------------------------- |
| `addCard(card)`                                   | `{shift, driver, crew, ...}` | Додає нове завдання для дати             |
| `removeCard(cardId)`                              | `cardId`                     | Видаляє завдання                         |
| `updateCard(cardId, updates)`                     | `cardId, updates`            | Оновлює завдання (змістить, екіпаж тощо) |
| `getBusyPersonnel(date, schedule)`                | `date, schedule`             | Повертає Set ID людей, які зайняті       |
| `getBusyPersonnelForTime(date, timeFrom, timeTo)` | `date, timeFrom, timeTo`     | Людей зайнятих у конкретний час          |
| `saveSchedule()`                                  | -                            | Зберігає дані в `localStorage`           |
| `copyFromPreviousDay(date)`                       | `date`                       | Копіює розклад попереднього дня          |
| `addPersonnel(person)`                            | `{name, role}`               | Додає персонал до списку                 |
| `removePersonnel(id)`                             | `id`                         | Видаляє персонал                         |
| `updatePersonnel(person)`                         | `person`                     | Оновлює дані персоналу                   |

### Ключові логіки:

**Міграція даних** — При завантаженні переводить старі ролі (`driver` → `crew_driver`, `local_crew` → `local`)

**Перевірка зайнятості** — Система автоматично запобігає призначенню однієї людини на кілька місць одночасно

## 🔷 `App.jsx` — Головний компонент

**Розташування:** `src/App.jsx`

**Призначення:** Головний контейнер додатку, управління станом та макетом.

**Основні функції:**

1. **Автентифікація** — Перевірка входу, показ LoginPage якщо не залогінений

```
App
├── LoginPage (якщо не залогінений)
└── Головна сторінка
    ├── Залізна смуга (Header)
    │   ├── Вибір дати
    │   ├── Статистика (DayStats)
    │   └── Кнопки управління
    ├── Управління персоналом (PersonnelManager)
    ├── Заголовок розпорядження (NaryadHeader)
```
